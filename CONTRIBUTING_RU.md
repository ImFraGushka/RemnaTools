# 🤝 Руководство по вкладу в RemnaTools

Спасибо за интерес к развитию RemnaTools! Этот документ содержит рекомендации для разработчиков.

## Структура кода

### Главный скрипт: `RWTools.sh`

Скрипт организован по логическим секциям:

#### Основная конфигурация
- `load_config()` - Загрузка настроек из `/opt/remnatools/config.conf`
- `save_config()` - Сохранение настроек в конфиг-файл
- Определения платформ: ассоциативный массив `PLATFORM_APPS`

#### Интерфейс пользователя
- `interactive_menu()` - Универсальная навигация с поддержкой стрелок
- `draw_platform_selection_menu()` - Выбор приложений для каждой платформы
- Все меню используют: ↑↓ стрелки, Space для выбора, Enter для подтверждения

#### Интеграция с API
- `get_existing_apps()` - Запрос к API панели с обработкой ошибок
- Обработка HTTP статус-кодов
- Fallback эндпоинты для совместимости

#### Установка инфраструктуры
- `auto_install_panel()` - Установка Caddy + PostgreSQL + Docker
- `auto_install_node()` - Установка VPN ноды с BBR3

#### Управление бэкапами
- `manage_backups()` - Главное меню бэкапов
- `create_backup_now()` - Создание tar.gz и загрузка в Telegram
- `configure_backup_auto()` - Настройка cron для автоматических бэкапов
- `restore_backup()` - Восстановление из архива

## Рекомендации по Bash

### 1. Используйте `local` для переменных функций:
```bash
my_function() {
    local var="value"
    local -a array=()
    local -A assoc=()
}
```

### 2. Правильное цитирование:
```bash
# Хорошо
echo "$var"
if [[ "$var" == "value" ]]; then

# Избегайте
echo $var
if [ $var = value ]; then
```

### 3. Обработка ошибок:
```bash
local response=$(curl -s ...)
if [ $? -ne 0 ]; then
    echo "Ошибка"
    return 1
fi
```

### 4. Используйте `[[ ]]` для условий:
```bash
# Хорошо
if [[ "$name" =~ ^[a-z]+$ ]]; then

# Менее предпочтительно
if [ -n "$name" ]; then
```

## Добавление новых функций

### 1. Добавьте новый пункт меню:
```bash
# В главный цикл меню
"8")
    my_new_function
    ;;
```

### 2. Создайте новую функцию:
```bash
my_new_function() {
    local param1="$1"
    local param2="$2"
    
    echo "Обработка..."
    # Реализация
    
    return 0
}
```

### 3. Добавьте поддержку конфигурации:
```bash
# В load_config()
MY_SETTING=${MY_SETTING:-"значение_по_умолчанию"}

# В save_config()
echo "MY_SETTING=$MY_SETTING" >> "$CONFIG_FILE"
```

```bash
# В load_config()

# В draw_platform_selection_menu() - автоматически определится
```

### Добавление API эндпоинта

```bash
get_new_data() {
    local url="$PANEL_URL/api/new/endpoint"
    local response=$(curl -s -H "Authorization: Bearer $API_TOKEN" "$url" -w "\n%{http_code}")
    local http_code=$(echo "$response" | tail -n1)
    local body=$(echo "$response" | head -n-1)
    
    if [ "$http_code" != "200" ]; then
        echo "Ошибка: HTTP $http_code"
        return 1
    fi
    
    echo "$body" | jq .
}
```

### Добавление Cron задачи

```bash
(crontab -l 2>/dev/null; echo "0 * * * * /opt/remnatools/task.sh") | crontab -
```

## Отчёты об ошибках

При отчёте об ошибках включите:

1. **ОС и версия:** `lsb_release -a`
2. **Версия RemnaTools:** Проверьте в скрипте
3. **Сообщение об ошибке:** Полный вывод
4. **Шаги воспроизведения**
5. **Ожидаемое vs фактическое поведение**
6. **Состояние системы:** `docker ps`, `docker-compose ps`

## Безопасность

При изменении скрипта:

- Никогда не хардкодьте токены или пароли
- Валидируйте все входные данные пользователя
- Используйте правильное цитирование для предотвращения injection
- Проверяйте HTTP статус-коды
- Реализуйте rate limiting для API вызовов
- Сохраняйте чувствительные данные с правильными permissions (600)
